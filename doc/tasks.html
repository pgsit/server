Tasks Workflow Documentation

This document describes all task-related endpoints and the end-to-end workflow for creating and viewing todos.

Overview

The tasks feature is built around a todos SQLite table. Users submit tasks via an HTML form and retrieve them through a set of filtered view and API endpoints.

Database Schema

CREATE TABLE todos (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    username    TEXT    NOT NULL,   -- name of the person
    description TEXT    NOT NULL,   -- multiline task description
    date        TEXT    NOT NULL,   -- YYYY-MM-DD
    time        TEXT    NOT NULL,   -- HH:MM
    created_at  TEXT    NOT NULL    -- RFC3339 timestamp
);

Endpoints

GET /todo  —  Task Creation Form

Renders an HTML form for creating a new task.

FieldWidgetNotes
NameText inputStored as username
Date & TimeDate-time pickerDefaults to current date/time
DescriptionMultiline textarea6 rows
All fields are required. Submitting with any field empty re-renders the form with an inline error message and preserves entered values.

POST /todo  —  Save Task

Accepts the form submission, validates all fields are non-empty, splits the datetime-local value into separate date and time columns, and inserts the record. On success, redirects to GET /todo?saved which shows a confirmation banner.

GET /tasks  —  Redirect

Issues a 301 Moved Permanently redirect to /tasks/view.

GET /tasks/view  —  HTML Task List

Renders a full HTML page listing tasks in a table. Supports a filter query parameter.

Filter valueURLReturns
all (default)/tasks/view or /tasks/view?filter=allAll tasks, ordered by date then time
today/tasks/view?filter=todayTasks for today, ordered by time
tomorrow/tasks/view?filter=tomorrowTasks for tomorrow, ordered by time
week/tasks/view?filter=weekTasks from today through the next 7 days

The page includes four center-aligned navigation buttons (All, Today, Tomorrow, Week) with the active filter highlighted. Past tasks (date before today) are rendered in green.

GET /tasks/today  —  JSON

Returns a JSON array of tasks scheduled for today, ordered by time.

GET /tasks/tomorrow  —  JSON

Returns a JSON array of tasks scheduled for tomorrow, ordered by time.

GET /tasks/weekly  —  JSON

Returns a JSON array of tasks between today and 7 days from now (inclusive), ordered by date then time.

JSON Response Shape

All JSON task endpoints return the same object structure. id and created_at are intentionally omitted from responses.

{
  "username":    "Alice",
  "description": "Prepare release notes",
  "date":        "2026-04-28",
  "time":        "14:00",
  "past":        false
}

The past field is true when the task's date is before today.

Workflow Summary

  1. User opens /todo in a browser — form pre-fills current date/time.
  2. User fills in Name, Description, and adjusts the date/time picker, then submits.
  3. Server validates all fields; on failure re-renders form with error; on success saves to DB and redirects back with a confirmation banner.
  4. User browses to /tasks/view (or /tasks) to see all tasks with filter buttons.
  5. Past tasks appear in green; active filter button is highlighted.
  6. Programmatic clients can query /tasks/today, /tasks/tomorrow, or /tasks/weekly for JSON responses.
